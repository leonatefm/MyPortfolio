    Michigan Engineering

    Web Designer & Developer, University of Michigan

    (Content Under Construction)

    This is a website called victors experience, which is a showcase of the amazing work being done by Michigan engineers. The target users are alumni who may be interested in those projects and are willing to give future donations to help engineers to make our world better. Since we are called to attract donations, the website must look fancy. Thus, it actually quite challenging when you design for money.

    This website is created mainly by two people. We designed it together and for the development, I’m primarily responsible for the front end thing while my colleague focusing on the back end developing. The most challenging part of this project for me is the design of the transition among different views. As you may notice, this website is a single page application. So all the transitions are enabled using javascript in the client side. The data is loaded using AJAX and deployed manually. Since the information of each item is the same in all the views, so the HTML structure actually stays the same while the css style changes a lot when you switch the view.

    One common issue of one-page website is the loss of go back and forward feature. Here I use the hash in the url to track the interaction history and fake a natural feature of going back or forward. It’s a pain, but we made it.
